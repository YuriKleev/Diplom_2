# Диплом. Задание №2: Автотесты для API сервиса Stellar Burgers

## Описание дипломного проекта

**Stellar Burgers** — это сервис, позволяющий собрать и заказать бургер из необычных ингредиентов.

В рамках дипломного проекта необходимо покрыть автотестами следующие эндпоинты API сервиса **Stellar Burgers**:

### **Создание пользователя**

- создание уникального пользователя;
- создание пользователя, который уже зарегистрирован;
- создание пользователя с одним незаполненным обязательным полем.

### **Логин пользователя**

- вход под существующим пользователем;
- вход с неверным логином и паролем.

### **Создание заказа**

- с авторизацией;
- без авторизации;
- с ингредиентами;
- без ингредиентов;
- с неверным хешем ингредиентов.

Автотесты по описанным выше сценариям выполнены с использованием **Pytest**, **Requests** и **Allure**.

### Структура репозитория с дипломным проектом

- `allure-results` - Allure-отчет
- `tests` - тесты
- `conftest.py` - фикстуры
- `data.py` - тестовые данные
- `endpoints.py` - эндпоинты
- `helper.py` - генератор случайных хешей ингредиентов
- `methods.py` - методы для запросов
- `status_codes.py` - коды состояния HTTP

### Запуск автотестов

#### **Запуск автотестов с созданием отчета о тестировании в Allure**

 `pytest ./tests --alluredir=./allure-results`

#### **Формирование отчета в Allure**

 `allure serve ./allure-results`
